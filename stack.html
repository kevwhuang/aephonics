Web Engineering Stack

aephonics · March 4, 2026

Foundation

  1. 01
    Astro Framework

    Static-first framework that outputs zero JavaScript by default. Interactive components hydrate in isolation as Astro islands, leaving the rest of the page untouched. Each route manages its own JS budget, which keeps performance predictable across projects of varying complexity. Cross-island state is handled with nanostores — framework-agnostic, under 1kb, and designed specifically for this architecture.

  2. 02
    TypeScript Language

    Used across every project without exception. Typed props, schemas, and API responses catch shape mismatches and broken references at compile time rather than in production. Astro supports TypeScript natively, and the coverage significantly improves output quality when working with AI coding tools.

Development

  1. 03
    Bun Runtime

    JavaScript runtime and package manager for local development. Faster installs and dev server startup than Node across the board. The full toolchain runs through it without modification. Production targets Node on Netlify, so Bun-specific APIs are kept out of application code.

  2. 04
    VS Code Editor

    Primary editor across all client work. Strong TypeScript intellisense, dedicated Astro and Tailwind extensions, and tight integration with the terminal-based toolchain. A consistent setup across projects removes friction when switching between engagements.

  3. 05
    Claude Code Editor

    AI coding agent running in the terminal with full codebase context. Handles file scaffolding, multi-file edits, and large refactors. TypeScript and Tailwind are the ideal pairing — typed interfaces and colocated styles give it accurate context to produce code that fits the existing codebase.

  4. 06
    GitHub Version Control

    Version control and remote repository host for all projects. Pull requests, Netlify branch previews, and Claude Code all operate directly from the repository. GitHub Actions handles CI/CD — automated checks run on every PR and block merges on failure. The full pipeline is defined in .github/workflows and version-controlled alongside the codebase.

Infrastructure

  1. 07
    Netlify Deployment

    Deployment and hosting for all Astro projects. Git-based deploys, automatic SSL, edge-distributed assets, and per-branch preview URLs on all plans. Netlify Functions handles any server-side requirements beyond Astro SSR. No infrastructure to provision or maintain.

  2. 08
    Supabase Backend

    Postgres database, auth, and file storage under one platform. Data is accessed through Astro SSR endpoints via the TypeScript client — no custom server required. Auth handles sessions and access control out of the box. The Claude Code MCP integration allows schema management and queries directly from the terminal during development.

UI

  1. 09
    Tailwind Styling

    Utility classes written directly in markup, colocated with the component they style. Extended per project with custom design tokens — brand colors, type scale, spacing. No separate stylesheet to maintain, no naming conventions to enforce, and styles are always visible in context when working with AI tools.

  2. 10
    React UI Language

    Used only where client-side state or interaction is required, scoped to Astro islands. The surrounding page stays as static HTML. This keeps JavaScript proportional to actual UI requirements rather than loading a full framework as the default.

  3. 11
    shadcn/ui Components

    Components are copied into the codebase rather than installed as a package. Built on Radix UI for accessibility and Tailwind for styling, consistent with the rest of the stack. Every component is fully editable and not subject to upstream breaking changes. Lucide ships as the default icon set — tree-shakeable and visually consistent with the shadcn design language.

Animation

  1. 12
    GSAP Animation

    Used for page-level animation — entrance sequences, staggered reveals, and multi-element timelines. Runs as a standard script in Astro with no React dependency. The timeline API gives precise, sequenced control over motion that CSS transitions can't replicate.

DX

  1. 13
    Prettier Tooling

    Opinionated code formatter applied across all file types — TypeScript, HTML, CSS, JSON. Formatting is automatic and consistent, with no manual decisions about style. Configured with eslint-config-prettier to disable any ESLint rules that would conflict.

  2. 14
    ESLint Tooling

    Static analysis across every project, configured with ESLint 9 flat config. Extended with typescript-eslint, eslint-plugin-astro, and eslint-plugin-react. Catches type misuse, dead code, and inconsistent patterns at save time in VS Code and enforced again in the pre-deploy pipeline.

Testing

  1. 15
    Vitest Unit Testing

    Unit and component testing built on Vite. Because Astro already uses Vite internally, Vitest requires minimal configuration and shares the same transform pipeline. TypeScript support is native and the API mirrors Jest, making it familiar to most JavaScript developers. Used for isolated logic, utility functions, and component behavior.

  2. 16
    Playwright E2E Testing

    End-to-end testing in real browsers. Covers critical user paths — form submissions, navigation, auth flows — against the actual running site. TypeScript-native and integrates directly with Netlify branch preview URLs, enabling E2E tests to run against every PR before it merges.
